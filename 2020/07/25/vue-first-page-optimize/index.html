vue如何优化首页的加载速度？首页的白屏是什么原因引起的？以及如何优化

SPA页面加载过程

1.拉取html(FP:html加载下来了)

2.拉取静态资源 css/js
这里的js如果没有任何处理的话，拉下来的js就非常大，因此造成之后的过程就非常慢
3.解析js，生成html代码(FCP：css/js静态资源加载下来了，这个过程非常快)

4.ajax请求 0.5s
解析的js里面发现有ajax请求，那么就去请求ajax
从这往上，全是白屏
5.数据填充(FMP:数据请求回来了)

指标

FP：第一帧像素落点
FCP：首次内容绘制
FMP：首次有效绘制

解决方式

1.骨架屏，在FP阶段加入页面的大结构，让用户提前看到页面的大体架构，SPA才会有白屏的问题，只有首页才会有白屏的问题
2.动态加载路由，提取公共库
路由懒加载：用到的时候再加载，不用不加载
可以把首页的js打到main.js里面，vue等库抽离出来，打成vue.js
首页加载的时候就只加载main.js和vue.js
切换路由的时候，会动态加载第二个页面的js，而此时vue.js已经有了，不会在下载了
用到的插件：babel-plugin-syntax-dynamic-import
路由懒加载写法：
•vue异步组件实现路由懒加载
component：resolve=>(['需要加载的路由的地址'，resolve])
•es提出的import(推荐使用这种方式)
const HelloWorld = （）=>import('需要加载的模块地址')

3.ssr 预渲染 同构
ssr:服务端渲染(php,jsp)
同构:一套代码，多端使用(服务端，客户端,vue react nuxt)
预渲染:预先渲染，webpack把vue打包出来的项目拿到无头浏览器中去执行，执行完之后，就获取到html页面的内容，之后再吐到html文件中，再把html放到CDN上，用户再去拉页面的时候，相当于把ajax请求拿到FP阶段执行
