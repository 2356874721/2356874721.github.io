重绘与回流

发表于 2020-07-25 更新于 2020-08-08 分类于 css

前端性能优化中经常会谈到重绘和回流这两个概念，首先我们来看下重绘和回流什么时候发生，以及重绘和回流是什么

页面渲染的流程

1.浏览器把html解析成一颗DOM树，根节点是document，DOM树上包含displsy:none这样的节点
2.DOM转换成render树，render树能够识别样式，每一个节点都有自己的样式，render树上已经没有了display:none这样的节点了
3.render树构建完毕，浏览器就根据渲染好的render数来渲染页面
reflow:当render tree中的一部分（或全部）因为元素的规模尺寸，布局，隐藏等改变而需要重新构建。这就称为回流（其实我觉得叫重新布局更简单明了些）。每个页面至少需要一次回流，就是在页面第一次加载的时候。
repaint:当render tree中的一些元素需要更新属性，而这些属性只是影响元素的外观、风格，而不会影响布局的，比如background-color。则就叫称为重绘。
通过上述定义，可以很明显看出，重绘的代价要比回流小，毕竟重绘只涉及样式的改变，不涉及到布局。回流必将引起重绘，而重绘不一定会引起回流。
那么什么操作会引起回流、重绘呢

引起回流、重绘的操作

任何对render tree中元素的操作都会引起回流或者重绘
•页面初始化布局
•添加、删除元素（回流+重绘)
•隐藏元素，display:none(回流+重绘)，visibility:hidden(只重绘，不回流)
•窗口大小变化，浏览器字体大小等
•对style的操作，如:padding margin border color fontsize等等
我们知道了引起回流和重绘的原因，那么怎么减少回流和重绘呢？

减少回流、重绘

•不要一个一个改变元素的样式属性，最好直接改变className，但className是预先定义好的样式，不是动态的，如果你要动态改变一些样式，则使用cssText来改变

•让要操作的元素进行”离线处理”，使用documentFragment或div等元素进行缓存操作，这个主要用于添加元素的时候，大家应该都用过，就是先把所有要添加到元素添加到1个div(这个div也是新加的)，最后才把这个div append到body中。
•频繁用到的DOM节点用变量进行缓存
•将复杂的元素绝对定位或固定定位，使它脱离文档流

原因

css盒模型

发表于 2020-07-23 更新于 2020-08-08 分类于 css

css中包含三种盒模型：标准盒模型、IE怪异盒模型、flex弹性盒模型

标准盒模型

标准和模型从内到外依次是：content/padding/border/margin
代码中设置的width/height是给content设置的
问题：写好的布局，要加一点内边距，会让盒子本身变大，如：初始时设置宽高各100px，加了10px内边距，此时盒子整个是120*120。所以怎么解决这个问题呢，可以通过IE怪异盒模型来解决

怪异盒模型

和标准和模型不同的是，怪异盒模型的width/height是给border(包含)里面设置的，如：宽高各100px，加了10px的内边距，整个盒子并不会变大，会把内容往里面挤
开启怪异盒模型的方式：box-sizing: border-box(标准盒模型值是content-box，还有一个值是inherit，来自继承的属性)

弹性盒模型

弹性盒模型主要是用于响应式页面
弹性盒子由弹性容器(Flex container)和弹性子元素(Flex item)组成
弹性容器通过设置 display 属性的值为 flex 或 inline-flex将其定义为弹性容器
弹性容器内包含了一个或多个弹性子元素
注意： 弹性容器外及弹性子元素内是正常渲染的。弹性盒子只定义了弹性子元素如何在弹性容器内布局
弹性子元素通常在弹性盒子内一行显示。默认情况每个容器只有一行
具体语法可以参看另一篇文章：http://www.sheriff.run/2020/07/20/css-flex/

inline-block元素间隔问题

发表于 2020-07-20 更新于 2020-08-08 分类于 css

产生间隔的原因

inline-block元素排版的时候，原来HTML代码中的回车换行**被转成一个空白符**，在字体不为0的情况下，空白符占据一定宽度，所以inline-block的元素之间就出现了空隙。

解决方案

•给父级元素设置font-size： 0；子元素设置相应的font-size

•改变书写方式，元素间留白间距出现的原因就是标签段之间的空格，因此，去掉HTML中的空格，自然间距就消失了

•在要转换为inline-block的元素之间加入注释符，将换行注释掉就ok了

•将结束标签与下一个开始标签写在一起，这样，他们之间的空格也会被清除掉

•将元素结束标签的”>“放在下一行的开始处

•使用word-spacing或letter-spacing

一个是字符间距(letter-spacing)一个是单词间距(word-spacing)，大同小异。letter-spacing子元素要设置letter-spacing为0，不然会继承父元素的值；使用word-spacing时，只需设置父元素word-spacing为合适值即可。

•使用margin负值

块级元素和行内元素

发表于 2020-07-20 更新于 2020-08-08 分类于 css

块级元素

特点

•独占一行显示

•可以设置宽高，宽度默认100%父元素，高度是内容撑开的

•可以设置margin，padding

常见的块级元素

行内元素

特点

•行内元素不会独占一行，相邻的行内元素会排在同一行

•不能设置宽高，宽高都是内容撑开的

•行内元素水平方向的margin-left; margin-right; padding-left; padding-right;可以生效。但是竖直方向的margin-bottom; margin-top; padding-top; padding-bottom;却不能生效

常见的行内元素

块级元素和行内元素相互转换

可以通过修改display属性来切换块级元素和行内元素

display: block可以把行内元素完全变成一个块级元素

display: inline可以把块级元素完全变成一个行内元素

display: inline-block既可以设置宽高，又可以和其他行内元素在一行

隐藏元素

发表于 2020-07-20 更新于 2020-08-08 分类于 css

隐藏元素

css中对元素进行隐藏有以下几种方式：

相同作用

能够对元素进行隐藏，但是他们是有区别的

区别

空间占据

•display: none;不占据空间，所以动态改变此属性时会引起重排

•visibility: hidden元素会被隐藏，但是不会消失，依然占据空间

•opacity=0只是透明度为100%,元素隐藏，依然占据空间

继承性

•display:none不会被子元素继承，但是父元素都不在了，子元素自然也就不会显示了

•visibility:hidden 会被子元素继承，可以通过设置子元素visibility:visible 使子元素显示出来

•opacity: 0 也会被子元素继承，但是不能通过设置子元素opacity: 0使其重新显示

动画效果transition

display

鼠标hover后没有效果

visibility

元素从隐藏到展示不能实现动画效果

如果是元素从显示到隐藏，则可以进行实现动画效果

opacity

对transition有效

事件触发

•display:none 无占用空间元素，因此无法点击事件

•opacity:0 时 元素不可显示 但是其事件有效

•visibility:hidden元素无效 绑定的事件无效:已经不可以接受js效果

注意：点击、hover事件不反应，但是能够操作dom元素的css的属性、样式等
