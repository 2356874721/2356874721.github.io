跨域解决方案

发表于 2020-07-26 更新于 2020-08-08 分类于 js

同源策略

浏览器同源策略，只要协议、域名、端口号，有一个不一致，就会产生跨域

浏览器不支持跨域的原因

主要是为了安全
1、cookie、localStorage不能支持跨域，假如登录了银行的网站，银行就给我的浏览器注入了一个cookie，我又访问了一个恶意网站，这个网站就有拿到我的cookie，冒充我登录银行
2、DOM元素不支持跨域 iframe，自己的页面嵌入了别人的页面，我如果可以操作iframe里面的元素的话，那么嵌入的网站的新户名、密码等信息就有可能会暴露
3、ajax请求，接口不能对外开放

为什么还要实现跨域

跨域是由前后端分离导致的，在smarty时代，前后端代码混在一起，因此部署的时候也只能部署到一台服务器上，使用的域名也相同，就不会有跨域问题。
前后端分离以后，代码虽然分离了，但是仍然可以把代码部署到同一台服务器，也不会造成跨域。
前后端分离，后端大多是业务逻辑等，前端是资源，因此为了更好的维护，使前端页面更快地加载出来，可以把前端单独部署到一台web服务器上，因此引出了跨域问题，目前大多数项目中都是跨域请求前后端分离项目中，前端部署到一台服务器，后端部署到一台服务器，肯定会产生跨域

html标签语义化

怎么理解html语义化标签，这是面试中经常会问的问题，那么html语义化到底是什么意思呢

html语义化

简单的来说就是用正确的标签做正确的事情
例如：段落用 p 标签，标题用 h 系列标签，边栏用 aside 标签，主要内容用 main 标签。

使用语义化标签的好处

•增强可读性，对开发者更友好，便于团队的开发和维护。
•在没有CSS的情况下也能较好地呈现网页的内容结构与代码结构。
•有利于 SEO ，可以让搜索引擎爬虫更好地获取到更多有效信息，搜索引擎的爬虫依赖于标签来确定上下文和各个关键字的权重，有效提升网页的搜索量。
•方便其他设备解析（如屏幕阅读器，盲人阅读器，移动设备），以意义的方式来渲染网页。

html5标签

HTML5新增了哪些语义化标签呢？

html5新特性

•绘画canvas
•用于媒介回放的 video和audio元素
•本地离线存储 localStorage 长期存储数据，浏览器关闭后数据不丢失
•sessionStorage的数据在浏览器关闭后自动删除
•表单控件，calendar、date、time、email、url、search
•新的技术webworker、websocket、Geolocation

html5常用的api

•页面可见性API——Page Visibility
•API全屏API——Full Screen
•访问摄像头和麦克风——getUserMedia API
•电池API——Battery API
•预加载——Link Prefetching
•监听屏幕旋转变化接口: ——orientationchange
•手机震动: ——window.navigator.vibrate(200)
•当前语言: ——navigator.language
•联网状态: ——navigator.onLine
•页面可编辑: ——contentEditable
•浏览器活跃窗口监听: ——window.onblur & window.onfocus

如何看待html5

•渐进增强
试想，如果HTML5否定了之前的HTML文档，各种大大小小的网站都要重写，那么HTML5带来的就不是惊喜而是惊吓！实际上，HTML5的一个核心的理念就是保持一切新特性与原有功能保持过渡。
•简单至上
简单至上，尽可能简化，简化了DOCTYPE和字符集声明;强化了HTML5API，使页面设计更加简单;以浏览器的原生能力代替复杂的JavaScript代码。

hmtl5和html的区别

•在文档类型声明上

由这两者对比可见：在文档声明上，html有很长的一段代码，并且很难记住这段代码，想必很多人都是靠工具直接生成的吧？而html5却是不同，只有简简单单的声明，这也方便人们的记忆，更加精简。

重绘与回流

发表于 2020-07-25 更新于 2020-08-08 分类于 css

前端性能优化中经常会谈到重绘和回流这两个概念，首先我们来看下重绘和回流什么时候发生，以及重绘和回流是什么

页面渲染的流程

1.浏览器把html解析成一颗DOM树，根节点是document，DOM树上包含displsy:none这样的节点
2.DOM转换成render树，render树能够识别样式，每一个节点都有自己的样式，render树上已经没有了display:none这样的节点了
3.render树构建完毕，浏览器就根据渲染好的render数来渲染页面
reflow:当render tree中的一部分（或全部）因为元素的规模尺寸，布局，隐藏等改变而需要重新构建。这就称为回流（其实我觉得叫重新布局更简单明了些）。每个页面至少需要一次回流，就是在页面第一次加载的时候。
repaint:当render tree中的一些元素需要更新属性，而这些属性只是影响元素的外观、风格，而不会影响布局的，比如background-color。则就叫称为重绘。
通过上述定义，可以很明显看出，重绘的代价要比回流小，毕竟重绘只涉及样式的改变，不涉及到布局。回流必将引起重绘，而重绘不一定会引起回流。
那么什么操作会引起回流、重绘呢

引起回流、重绘的操作

任何对render tree中元素的操作都会引起回流或者重绘
•页面初始化布局
•添加、删除元素（回流+重绘)
•隐藏元素，display:none(回流+重绘)，visibility:hidden(只重绘，不回流)
•窗口大小变化，浏览器字体大小等
•对style的操作，如:padding margin border color fontsize等等
我们知道了引起回流和重绘的原因，那么怎么减少回流和重绘呢？

减少回流、重绘

•不要一个一个改变元素的样式属性，最好直接改变className，但className是预先定义好的样式，不是动态的，如果你要动态改变一些样式，则使用cssText来改变

•让要操作的元素进行”离线处理”，使用documentFragment或div等元素进行缓存操作，这个主要用于添加元素的时候，大家应该都用过，就是先把所有要添加到元素添加到1个div(这个div也是新加的)，最后才把这个div append到body中。
•频繁用到的DOM节点用变量进行缓存
•将复杂的元素绝对定位或固定定位，使它脱离文档流

原因

因为css中的1px并不等于设备的1px，这是因为不同的设备，像素比是不一样的。我们可以使用window.devicePixelRatio来获取像素比，计算公式为：像素比 = 设备物理像素/css逻辑像素
通常iphone中的视网膜屏，像素比为2或3，所以当你在css中写1px时，实际上设备上的像素已经是2或3了，因此边框看起来变粗了。

css盒模型

发表于 2020-07-23 更新于 2020-08-08 分类于 css

css中包含三种盒模型：标准盒模型、IE怪异盒模型、flex弹性盒模型

标准盒模型

标准和模型从内到外依次是：content/padding/border/margin
代码中设置的width/height是给content设置的
问题：写好的布局，要加一点内边距，会让盒子本身变大，如：初始时设置宽高各100px，加了10px内边距，此时盒子整个是120*120。所以怎么解决这个问题呢，可以通过IE怪异盒模型来解决

怪异盒模型

和标准和模型不同的是，怪异盒模型的width/height是给border(包含)里面设置的，如：宽高各100px，加了10px的内边距，整个盒子并不会变大，会把内容往里面挤
开启怪异盒模型的方式：box-sizing: border-box(标准盒模型值是content-box，还有一个值是inherit，来自继承的属性)

弹性盒模型

弹性盒模型主要是用于响应式页面
弹性盒子由弹性容器(Flex container)和弹性子元素(Flex item)组成
弹性容器通过设置 display 属性的值为 flex 或 inline-flex将其定义为弹性容器
弹性容器内包含了一个或多个弹性子元素
注意： 弹性容器外及弹性子元素内是正常渲染的。弹性盒子只定义了弹性子元素如何在弹性容器内布局
弹性子元素通常在弹性盒子内一行显示。默认情况每个容器只有一行
具体语法可以参看另一篇文章：http://www.sheriff.run/2020/07/20/css-flex/
